# Tammam Properties — Deployment Guide\n\n## Quick Deploy to Vercel (One-Click Alternative)\n\n### Option 1: Deploy Button (Fastest)\nClick this button to deploy directly to Vercel (automatically clones repo and prompts for env vars):\n\n[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/tammamaboahmad1-jpg/Tammam-Properties-for-Sale-and-Rent)\n\n### Option 2: Manual Vercel Deploy\n1. Go to https://vercel.com/new\n2. Sign in with GitHub\n3. Import this repository: tammamaboahmad1-jpg/Tammam-Properties-for-Sale-and-Rent\n4. **Before clicking Deploy**, add Environment Variables:\n   - NEXT_PUBLIC_SUPABASE_URL = (from Supabase project settings → API → URL)\n   - NEXT_PUBLIC_SUPABASE_ANON_KEY = (from Supabase → API → anon key)\n   - SUPABASE_SERVICE_ROLE_KEY = (from Supabase → API → service_role key) [Mark as Secret]\n   - ADMIN_SECRET = (any strong secret, e.g., \"my-super-secret-admin-2024\") [Mark as Secret]\n   - NEXT_PUBLIC_ADMIN_SECRET = (same or different value)\n   - NEXT_PUBLIC_ADMIN_WHATSAPP = +201234567890\n   - NEXT_PUBLIC_ADMIN_EMAIL = your@email.com\n   - OPENAI_API_KEY = (optional, for AI text generation)\n5. Click **Deploy** and wait for build to complete\n6. Copy the URL provided (e.g., https://tammam-properties-xxx.vercel.app)\n\n### Supabase Setup (Required for full features)\n1. Go to https://app.supabase.com → Create new project\n2. Copy values from Project Settings → API:\n   - Project URL → NEXT_PUBLIC_SUPABASE_URL\n   - anon key → NEXT_PUBLIC_SUPABASE_ANON_KEY\n   - service_role key → SUPABASE_SERVICE_ROLE_KEY\n3. Go to SQL Editor and run the entire contents of `schema.sql` from this repo\n4. Go to Storage → Create bucket named `property-images` (set to Public for easy access)\n5. Insert sample admin user (SQL):\n   ```sql\n   insert into users (id, name, email, role) values (gen_random_uuid(), 'Admin', 'admin@example.com', 'admin');\n   ```\n\n### Features & Routes\n- `/` — Home page (list of published properties)\n- `/add` — Public form to submit property listings (status=pending by default)\n- `/admin` — Admin dashboard to review and approve/reject submissions\n- `/properties/:id` — Property detail page with WhatsApp contact button\n- `/joke` — Random joke generator (works without Supabase)\n\n### Testing Locally Before Deploy\n```bash\ngit clone https://github.com/tammamaboahmad1-jpg/Tammam-Properties-for-Sale-and-Rent.git\ncd Tammam-Properties-for-Sale-and-Rent\ncp .env.example .env.local\n# Edit .env.local with your Supabase values\nnpm install\nnpm run dev\n# Visit http://localhost:3000/joke (works immediately)\n# Visit http://localhost:3000/add to test property submission\n```\n\n### API Endpoints\n- `POST /api/properties` — Submit new property listing\n- `POST /api/admin/properties/:id/approve` — Approve and publish (requires x-admin-secret header)\n- `POST /api/admin/properties/:id/reject` — Reject listing (requires x-admin-secret header)\n- `GET /api/joke` — Get random joke (external API)\n\n### Troubleshooting\n- **Build fails on Vercel**: Check that all env vars are added correctly (Settings → Environment Variables)\n- **\"Unauthorized\" on approve**: Ensure ADMIN_SECRET in Vercel matches the x-admin-secret header being sent\n- **No properties shown in /admin**: Submit a property via /add first (it will be status=pending)\n- **Images not uploading**: Verify bucket `property-images` exists in Supabase Storage and is public\n\n### Support\nFor issues or questions, refer to the main README.md or check GitHub Issues.\n"
